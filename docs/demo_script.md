# Сценарий демо

1. Открыть `http://localhost:5173`.
2. Войти как `user@tolmach.local` / `user123`.
3. Нажать `Новый запрос`.
4. Отправить: `покажи выручку по топ-10 городам за последние 30 дней`.
5. Показать ответ ассистента: трактовка, SQL, таблица, столбчатый график.
6. Нажать `Сохранить отчёт`, затем `Настроить рассылку`.
7. Переключиться между чатами в истории.
8. Отправить опасный запрос: `DROP TABLE users` и показать блокировку.
9. Выйти и войти как `admin@tolmach.local` / `admin123`.
10. Открыть `Логи` или `/admin/logs`.
11. Кликнуть строку и показать полный промпт, raw-ответ модели и статус.

## Акценты для защиты

- Пользователь работает на русском и не знает SQL.
- Чаты и сообщения сохраняются в PostgreSQL.
- Роли разделены: обычный пользователь не видит админские логи.
- Guardrails срабатывают до выполнения SQL.
- Text-to-SQL идёт через локальную Ollama `qwen3:4b`; если Ollama недоступна, backend пишет это в raw-ответ и использует deterministic fallback для демо.
